## ADR-003 : Base aliments curatée (300-500) vs API externe (50 000+)

**Date** : 2026-04-02
**Statut** : Accepté

### Contexte

Thirty a besoin d'une base de données d'aliments avec un profil microbiote. Deux stratégies sont possibles : curer manuellement une petite base de qualité, ou s'appuyer sur une API externe (Open Food Facts, USDA FoodData Central) pour couvrir un maximum d'aliments.

### Options évaluées

**Option A — API externe (Open Food Facts, USDA)**
- (+) Couverture massive : 50 000+ aliments immédiatement disponibles
- (+) Scan de code-barres possible day one
- (+) Moins de travail de curation initial
- (-) Ces bases ne contiennent PAS de scoring microbiote — elles ont des macros (calories, protéines, glucides) et parfois des fibres totales, mais pas de score prébiotiques, polyphénols, ou probiotiques
- (-) Pas de notion de mode de préparation avec impact microbiote
- (-) Beaucoup de bruit : 50 000 produits transformés dont la majorité n'a aucun intérêt microbiote
- (-) Dépendance à une API tierce (latence, disponibilité, changements de format)
- (-) Le mapping "données nutritionnelles classiques → score microbiote" serait un exercice d'approximation hasardeux

**Option B — Base curatée maison (300-500 aliments)**
- (+) Chaque aliment a un profil microbiote complet et vérifié : fibres solubles/insolubles, prébiotiques, polyphénols, probiotiques, oméga-3, soutien muqueux
- (+) Chaque aliment a des modificateurs par mode de préparation — la feature différenciante de Thirty
- (+) Qualité > quantité : 300 aliments bien scorés couvrent 90%+ des repas courants en France
- (+) Pas de dépendance externe
- (+) Le travail de curation EST la valeur du produit — c'est le "moat"
- (-) Travail initial conséquent (estimation : 2-3 semaines de recherche/curation)
- (-) L'utilisateur peut ne pas trouver un aliment spécifique → frustration
- (-) Maintenance manuelle pour ajouter de nouveaux aliments

### Décision

**Base curatée de 300-500 aliments au MVP, avec possibilité d'ajout libre par l'utilisateur.**

La valeur de Thirty n'est pas d'être un annuaire alimentaire exhaustif — c'est le scoring microbiote par aliment et par mode de préparation. Aucune API externe ne fournit cette donnée. Il faut la construire.

300 aliments bien choisis couvrent la quasi-totalité des repas quotidiens :
- ~80 légumes courants
- ~40 fruits
- ~30 légumineuses et céréales
- ~25 noix, graines
- ~30 herbes, épices
- ~30 produits laitiers et fromages
- ~30 viandes et poissons
- ~20 condiments fermentés et autres
- ~15-20 aliments inclassables courants

L'ajout libre (avec catégorisation manuelle par l'utilisateur) sert de soupape pour les aliments non couverts, et les aliments libres les plus populaires seront prioritaires pour être ajoutés à la base officielle.

### Conséquences

- Le package `@thirty/food-db` contient le seed data en JSON/YAML avec sources bibliographiques
- Chaque aliment a un champ `sources[]` listant les études/références utilisées pour le scoring
- Un mécanisme d'ajout libre est nécessaire dès le MVP avec une catégorisation simplifiée
- Phase 2 : intégration Open Food Facts en complément (pas en remplacement) pour les produits transformés
- Le travail de curation sera documenté dans un guide méthodologique (`docs/food-scoring-methodology.md`)
