## ADR-007 : Coolify sur VPS Hetzner vs Fly.io vs VPS nu

**Date** : 2026-04-02
**Statut** : Accepté

### Contexte

Thirty a besoin d'un environnement de production pour le MVP : une app NestJS, une app React (statique ou SSR), et une base PostgreSQL. Le budget est limité (projet perso), et l'infra doit montrer une approche raisonnée à un recruteur.

### Options évaluées

**Option A — Fly.io**
- (+) Excellent DX : `fly deploy` et c'est en prod
- (+) PostgreSQL managé (Fly Postgres)
- (+) SSL automatique, scaling horizontal facile
- (+) Bonne réputation dans la communauté indie/startup
- (-) Pricing opaque et incidents de facturation rapportés par la communauté
- (-) Fly Postgres n'est pas un vrai managed DB (c'est un wrapper, pas de backups automatiques fiables par défaut)
- (-) Vendor lock-in sur le format de déploiement (fly.toml)
- (-) Coût estimé MVP : ~15-25€/mois (app + DB + bandwidth)

**Option B — VPS nu (Hetzner / OVH)**
- (+) Le moins cher : Hetzner CX22 à ~4€/mois
- (+) Contrôle total
- (+) Pas de vendor lock-in
- (-) Tout à configurer à la main : Docker Compose, Nginx/Caddy, SSL, CI/CD, backups, monitoring
- (-) Temps de setup initial conséquent (1-2 jours)
- (-) Maintenance ongoing (updates OS, sécurité, certificats)

**Option C — Coolify sur VPS Hetzner**
- (+) Interface PaaS self-hosted : push-to-deploy via GitHub webhooks
- (+) SSL automatique (Let's Encrypt intégré)
- (+) PostgreSQL déployable en un clic dans Coolify
- (+) Dashboard de monitoring basique inclus
- (+) Coût = prix du VPS uniquement (~4-6€/mois)
- (+) Pas de vendor lock-in : c'est du Docker sous le capot, migrable n'importe où
- (+) Montre à un recruteur qu'on sait déployer sans PaaS magique tout en restant productif
- (-) Coolify est un projet open-source relativement jeune (mais actif, v4 stable)
- (-) Si Coolify a un bug, c'est sur toi (pas de support payant)
- (-) Un seul serveur = pas de haute disponibilité (acceptable pour un MVP)

### Décision

**Coolify sur VPS Hetzner (CX22 : 2 vCPU, 4 Go RAM, 40 Go SSD, ~4€/mois).**

C'est le meilleur ratio productivité/coût/contrôle pour un MVP :
- Le DX est comparable à Fly.io (push to deploy, SSL auto)
- Le coût est 3-5× inférieur
- L'infra est entièrement sous contrôle et migrable
- L'expérience self-hosting existe déjà (Vaultwarden)

La haute disponibilité n'est pas un sujet pour un MVP avec quelques dizaines d'utilisateurs. Si Thirty prend de l'ampleur, la migration vers un setup multi-serveur ou un PaaS managé sera un exercice documenté (et un bon sujet d'ADR futur).

### Conséquences

- Le setup initial inclut : installer Coolify sur le VPS, connecter le repo GitHub, configurer les services (API, front, PostgreSQL)
- Le front React est buildé en statique et servi par Coolify (ou un Nginx intégré)
- Les backups PostgreSQL doivent être configurés manuellement (cron + pg_dump vers un stockage externe, ex: Hetzner Object Storage à ~0.005€/Go)
- Le domaine `thirty.app` (ou similaire) pointe vers le VPS via un A record
- Les variables d'environnement sont gérées dans Coolify (pas de .env en prod)
- Un `docker-compose.yml` de développement local mirror la topologie de prod
